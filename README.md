# Distribution-shift-aware safe offline RL for regional integrated energy dispatch

This repository contains the code, configuration files, processed benchmark data, figures and LaTeX manuscript sources for an Applied Energy-style study on physics-constrained safe offline reinforcement learning for regional integrated electricity--heat system dispatch under distribution shift.

Public repository: <https://github.com/hepei002/safe-offline-rl-ries-dispatch>

The project is a public-data-driven simulation benchmark. It does not contain proprietary measurements from a real industrial park.

## What is included

```text
configs/                 System, dataset, shift, RL and safety-layer configurations
src/                     Reusable Python modules
scripts/                 Data processing, experiment and plotting entry points
tests/                   Unit/regression tests for core modelling components
data/processed/          Small derived benchmark tables and metadata intended for reproducibility
docs/                    Data provenance, model equations, experiment logs and release notes
results/                 CSV summaries and manuscript-ready figures, excluding large TIFF files
manuscript/latex/        Elsevier/Applied Energy LaTeX manuscript source
```

The repository intentionally excludes large third-party raw data files, intermediate data, local build folders, Python caches and high-resolution TIFF exports. See [data/README.md](data/README.md) and [docs/reproducibility_guide.md](docs/reproducibility_guide.md) for the reproduction workflow.

## Data policy

Raw OPSD and When2Heat files are third-party datasets and are not relicensed by this repository. Download them from the original providers using the links and version records in [docs/data_provenance.md](docs/data_provenance.md) and [docs/data_download_guide.md](docs/data_download_guide.md).

The processed hourly benchmark in `data/processed/` is a derived research artifact generated by the scripts in this repository. Before formal journal submission, archive the exact release snapshot on Zenodo, Figshare, Mendeley Data, OSF or an institutional repository and add the release DOI/accession to the manuscript and repository metadata.

## Quick start

Create a Python environment and install the core dependencies:

```powershell
python -m pip install -r requirements.txt
```

Run the test suite:

```powershell
python -m pytest tests
```

Regenerate the main experiment artifacts in the same order used by the manuscript:

```powershell
python scripts\plot_system_profiles.py
python scripts\run_deterministic_dispatch.py
python scripts\generate_offline_dataset.py
python scripts\run_shift_protocol_ood.py
python scripts\run_fixed_safety_projection.py
python scripts\run_adaptive_safety_mechanism.py
python scripts\run_week13_14_experiments.py
python scripts\run_week15_16_experiments.py
python scripts\make_enhanced_manuscript_figures.py
```

Compile the manuscript from `manuscript/latex/`:

```powershell
pdflatex -interaction=nonstopmode -halt-on-error main.tex
bibtex main
pdflatex -interaction=nonstopmode -halt-on-error main.tex
pdflatex -interaction=nonstopmode -halt-on-error main.tex
```
