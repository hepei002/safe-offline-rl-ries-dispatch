# 第六周：Gymnasium 环境接口与一致性说明

本周将前面建立的电–热综合能源系统包装为 Gymnasium 环境 `IntegratedEnergyEnv`，用于后续离线强化学习数据生成、策略评估和安全投影测试。环境代码位于 `src/envs/integrated_energy_env.py`，配置说明位于 `configs/env.yaml`。

## 1. 环境定位

`IntegratedEnergyEnv` 是一个小时级、单园区、连续动作环境。智能体只决定主要设备 set-point；电网购售电、弃风弃光、失负荷和热倾倒由环境根据电热平衡自动计算。因此，环境始终返回物理平衡后的调度结果，同时记录安全成本。

该设计有两个好处：

1. RL 策略不需要直接学习电热平衡的全部代数变量，动作维度较低。
2. 后续安全投影可以把 RL 候选动作投影到同一套设备 set-point 空间，再由环境统一结算。

## 2. 动作空间

动作空间为 `Box(low=-1, high=1, shape=(4,))`。

| 序号 | 名称 | 归一化动作 | 物理映射 |
|---:|---|---|---|
| 0 | `chp_electric_fraction` | \(a_0\in[-1,1]\) | \(P^{chp}=0.5(a_0+1)\bar P^{chp}\) |
| 1 | `heat_pump_heat_fraction` | \(a_1\in[-1,1]\) | \(H^{hp}=0.5(a_1+1)\bar H^{hp}\) |
| 2 | `battery_power_fraction` | \(a_2\in[-1,1]\) | \(a_2>0\) 为充电，\(a_2<0\) 为放电 |
| 3 | `thermal_power_fraction` | \(a_3\in[-1,1]\) | \(a_3>0\) 为储热充热，\(a_3<0\) 为储热放热 |

电池和储热动作会被 SOC 上下界硬裁剪，避免一步动作直接导致 SOC 越界。

## 3. 状态空间

观测空间为 10 维连续向量。

| 序号 | 名称 | 归一化规则 |
|---:|---|---|
| 0 | `electric_load_norm` | 当前电负荷 / 目标电峰值 |
| 1 | `heat_load_norm` | 当前热负荷 / 目标热峰值 |
| 2 | `pv_available_norm` | 当前光伏可用功率 / 光伏装机 |
| 3 | `wind_available_norm` | 当前风电可用功率 / 风电装机 |
| 4 | `cop_norm` | 当前热泵 COP / 5 |
| 5 | `import_price_norm` | 当前购电价格 / 200 |
| 6 | `battery_soc_norm` | 电池能量 / 电池容量 |
| 7 | `thermal_soc_norm` | 储热能量 / 储热容量 |
| 8 | `hour_sin` | \(\sin(2\pi h/24)\) |
| 9 | `hour_cos` | \(\cos(2\pi h/24)\) |

环境不会在观测中提供未来负荷、未来可再生出力或未来价格；滚动 MPC 和离线 RL 轨迹生成如果需要预测信息，应在策略侧单独构造。

## 4. 状态转移与平衡变量

给定动作后，环境先计算 CHP、热泵、电池和储热的实际动作，再更新 SOC：

\[
E^b_{t+1}=(1-\lambda^b)E^b_t+\eta^{bc}P^{bc}_t-\frac{P^{bd}_t}{\eta^{bd}},
\]

\[
E^h_{t+1}=(1-\lambda^h)E^h_t+\eta^{hc}H^{tc}_t-\frac{H^{td}_t}{\eta^{hd}}.
\]

热平衡不足部分记为热失负荷，热过剩部分记为热倾倒：

\[
S^h_t=\max(L^h_t-H^{supply}_t,0),\quad
D^h_t=\max(H^{supply}_t-L^h_t,0).
\]

电侧由电网购售电、弃风弃光和电失负荷自动平衡。电网容量不足时，剩余缺口记为电失负荷；供给过剩时，优先上网，超过上网容量后记为弃能。

## 5. 奖励与安全成本

奖励为单小时运行成本的相反数：

\[
r_t=-C_t.
\]

成本包括购电、燃气、弃能惩罚、失负荷惩罚和上网收益，口径与第五周行为策略评价一致。

`info["safety_cost"]` 记录：

\[
S^e_t+S^h_t+D^h_t+C^{pv}_t+C^{wind}_t.
\]

其中热倾倒和弃风弃光不一定代表不可行，但代表动作质量下降；电/热失负荷代表更严重的安全风险。

## 6. 硬执行约束与软记录约束

硬执行：

- CHP、热泵、电池、储热容量边界。
- 电池和储热 SOC 上下界。
- 电网购电和上网容量。
- 电热平衡残差为数值零。

软记录：

- 电失负荷。
- 热失负荷。
- 热倾倒。
- 弃风弃光。

这种划分使环境不会因为随机动作直接崩溃，同时保留安全成本供约束 RL、离线安全 RL 和安全投影模块使用。

## 7. 终止条件

默认 episode 长度为 24 h。执行 `episode_hours` 步后，`terminated=True`，`truncated=False`。当前环境不设置随机截断；后续若加入缺测、设备故障或极端事件中断，可通过 `truncated=True` 扩展。

## 8. 与优化器的一致性

测试 `tests/test_integrated_energy_env.py` 验证了三件事：

1. `reset()` 返回有限且位于观测空间内的归一化状态。
2. 将确定性优化器第一小时调度动作编码为环境动作后，环境一步转移得到的 SOC、电网功率和电热平衡残差与优化器一致。
3. 随机动作压力测试不会产生 NaN，且 reset 后不会泄漏上一条轨迹状态。

这满足第六周验收标准：相同输入和动作下，优化模型与环境状态转移一致；所有状态和动作均有明确归一化规则。
