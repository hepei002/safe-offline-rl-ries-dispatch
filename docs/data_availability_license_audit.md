## Current manuscript wording

The manuscript now states that the study uses public third-party datasets rather than measured proprietary park data, identifies each source version, notes that ERA5 is not used in the reported benchmark, and marks the derived benchmark/code repository DOI as a required pre-submission field.

## Required repository action before formal submission

Deposit the following files in a durable repository such as Mendeley Data, Zenodo, Figshare, OSF or an institutional repository:

- `data/processed/exogenous/germany_joint_hourly_2017_2019.csv`
- `data/processed/metadata/quality_report_2017_2019.json`
- source data behind manuscript figures and tables
- experiment configuration files
- preprocessing and experiment scripts
- README with variable definitions, units, source mapping, missing-value handling and licence notes

Do not claim that raw third-party data are newly generated by the authors. Prefer linking readers to the original OPSD and When2Heat records for raw downloads.

## Pre-submission risk flags

- The manuscript still contains `[repository DOI/accession to be inserted before submission]`. This must be replaced before final submission.
- Raw OPSD data aggregate multiple primary sources. The repository README should preserve source attribution and should not apply a new open licence to third-party raw data.
- When2Heat data can be attributed under CC BY 4.0, but its scripts use MIT. Keep data and code licences separate.
- ERA5 should not be mentioned as a used dataset unless future experiments actually use ERA5 fields.
